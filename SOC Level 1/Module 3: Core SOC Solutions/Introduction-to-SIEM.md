## Notes

### What is SIEM?

- **SIEM (Security Information and Event Management)** centralizes security logs from multiple devices.
- Enables analysts to:
  - Collect logs
  - Normalize data
  - Correlate events
  - Detect threats
  - Investigate alerts
- Core technology used by Security Operations Centers (SOCs).

### Types of Log Sources

#### Host-Centric Logs
Generated by endpoints and servers.

Examples:
- User logins
- File access
- Process execution
- Registry changes
- PowerShell activity

#### Network-Centric Logs
Generated by network infrastructure.

Examples:
- SSH connections
- FTP transfers
- Web traffic
- VPN connections
- Firewall events
- IDS/IPS alerts

---

### Challenges Without SIEM

- Logs are spread across many devices
- No centralized visibility
- Individual logs provide limited context
- Manual analysis is slow and inefficient
- Different log formats increase complexity

---

### Core SIEM Features

#### Centralized Log Collection
- Collects logs from endpoints, servers, firewalls, and network devices
- Uses agents, APIs, or forwarding methods

#### Log Normalization
- Converts different log formats into a standardized structure
- Makes searching and analysis easier

**Key Terms**
- **Parsing:** Breaking logs into individual fields
- **Normalization:** Converting different log formats into one consistent format

#### Log Correlation
- Combines events from multiple sources
- Identifies attack patterns that isolated logs would miss

Example:
- VPN login
- File access
- PowerShell execution
- Outbound network connection

Together these events may indicate data exfiltration.

#### Real-Time Alerting
- Detection rules monitor incoming events
- Alerts trigger automatically when rule conditions are met

#### Dashboards & Reporting
Provide visibility into:
- Triggered alerts
- Failed login attempts
- Event ingestion
- System health
- Top visited domains
- Security trends

- `/var/log/httpd`
- `/var/log/apache`
- `/var/log/cron`

#### Web Servers
Typically store:
- HTTP requests
- HTTP responses
- Error logs

### Log Ingestion Methods

- Agent / Forwarder
- Syslog
- Manual Upload
- Port Forwarding

---

### Detection Rules

SIEM detects threats using predefined or custom rules.

Examples:
- Multiple failed logins
- Successful login after repeated failures
- USB device insertion
- Large outbound data transfers

Example Windows Detection Rules:

**Event ID 104**
- Event logs cleared
- Potential attacker covering tracks

**Event ID 4688**
- Process creation
- Detect suspicious commands like `whoami`

---

### Alert Investigation

After an alert is generated, analysts determine whether it is:

#### False Positive
- Benign activity
- Detection rule may require tuning

#### True Positive
Possible response actions:
- Continue investigation
- Contact asset owner
- Isolate infected host
- Block malicious IP
- Begin incident response

## Conclusion

SIEM platforms are the foundation of modern Security Operations Centers. By centralizing, normalizing, and correlating logs from multiple sources, they provide analysts with the visibility needed to rapidly detect, investigate, and respond to security incidents.
